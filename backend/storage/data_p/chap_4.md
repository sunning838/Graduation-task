# 4과목 프로그래밍 언어 활용

## 1. 운영체제 (Operating System) 기초
- **개념**: 컴퓨터 시스템의 자원(하드웨어, 소프트웨어)을 효율적으로 관리하고, 사용자가 컴퓨터를 편리하고 효과적으로 사용할 수 있도록 환경을 제공하는 시스템 소프트웨어.
- **운영체제의 목적**: 처리능력(Throughput) 향상, 반환시간(Turn Around Time) 단축, 사용 가능도(Availability) 향상, 신뢰도(Reliability) 향상.
- **운영체제 핵심 기능**: 프로세스 관리, 주기억장치 관리, 파일 관리, 입출력 관리.

### 프로세스 (Process) 및 스레드 (Thread)
- **프로세스**: 주기억장치에 적재되어 실행 중인 프로그램. 프로세스 제어 블록(PCB)을 가진 프로그램.
- **프로세스 상태 전이**: 생성(New) → 준비(Ready) → 실행(Running) → 대기(Wait/Block) → 종료(Terminated).
  - **디스패치(Dispatch)**: 준비 상태에서 실행 상태로 전이.
  - **타이머 런아웃(Timer Run-out)**: 할당 시간이 종료되어 실행에서 준비 상태로 전이.

- **스레드**: 프로세스 내에서 실행되는 흐름의 단위. 프로세스의 자원을 공유하면서 독립적인 스케줄링을 받는다.

### 프로세스 스케줄링 (Scheduling)
- **선점형 (Preemptive)**: 한 프로세스가 CPU를 차지하고 있을 때, 우선순위가 높은 다른 프로세스가 현재 프로세스를 중단시키고 CPU를 빼앗을 수 있는 방식. (빠른 응답 시간, 오버헤드 발생)
  - **RR (Round Robin)**: 시간 할당량(Time Slice)을 정하여 그 시간만큼만 순서대로 실행.
  - **SRT (Shortest Remaining Time)**: 남은 처리 시간이 가장 짧은 프로세스에게 CPU 할당.
  - **다단계 큐 (Multi-level Queue)**: 작업들을 여러 종류의 그룹으로 나누어 여러 개의 큐를 이용.
- **비선점형 (Non-Preemptive)**: 한 프로세스가 CPU를 할당받으면 작업 종료 후 CPU 반환 시까지 다른 프로세스가 CPU를 점유할 수 없는 방식.
  - **FCFS (First Come First Served)**: 도착한 순서대로 CPU 할당.
  - **SJF (Shortest Job First)**: 실행 시간이 가장 짧은 프로세스에게 먼저 CPU 할당.
  - **HRN (Highest Response-ratio Next)**: 대기 시간과 서비스 시간을 이용해 우선순위 계산. `(대기시간 + 서비스시간) / 서비스시간`

### 교착상태 (Deadlock)
- **개념**: 둘 이상의 프로세스가 서로 다른 프로세스가 차지하고 있는 자원을 요구하며 무한정 기다리는 현상.
- **발생 4필요 조건**: 상호배제(Mutual Exclusion), 점유와 대기(Hold & Wait), 비선점(Non-preemption), 환형 대기(Circular Wait).
- **해결 기법**:
  - **예방 (Prevention)**: 4가지 조건 중 하나를 부정.
  - **회피 (Avoidance)**: 교착상태 가능성을 배제하지 않고 적절히 피해감. (예: **은행원 알고리즘(Banker's Algorithm)**)
  - **발견 (Detection)** 및 **회복 (Recovery)**.

## 2. 메모리 관리 (Memory Management)
- **가상 메모리 (Virtual Memory)**: 보조기억장치의 일부를 주기억장치처럼 사용하여 용량이 큰 프로그램을 실행할 수 있게 하는 기법.
  - **페이징 (Paging)**: 프로그램을 **동일한 크기**로 나눈 단위(페이지)로 관리. 내부 단편화 발생.
  - **세그먼테이션 (Segmentation)**: 프로그램을 **가변적인 크기**의 논리적 단위(세그먼트)로 나눈 단위로 관리. 외부 단편화 발생.

### 페이지 교체 알고리즘
- 주기억장치에 빈 프레임이 없을 때, 어떤 페이지를 쫓아낼지 결정하는 기법.
- **FIFO (First In First Out)**: 가장 먼저 들어와서 가장 오래 있었던 페이지 교체.
- **LRU (Least Recently Used)**: 최근에 가장 오랫동안 사용하지 않은 페이지 교체.
- **LFU (Least Frequently Used)**: 사용 빈도(횟수)가 가장 적은 페이지 교체.
- **OPT (Optimal)**: 앞으로 가장 오랫동안 사용하지 않을 페이지 교체 (이론상 최적).

## 3. 네트워크 (Network) 및 인터넷
- **OSI 7계층 (Open System Interconnection)**: ISO에서 제안한 통신망을 통한 상호 연결에 필요한 표준 아키텍처.
  1. **물리 계층**: 실제 비트(0과 1) 전송. (장비: 허브, 리피터)
  2. **데이터링크 계층**: 인접한 노드 간의 신뢰성 있는 전송, MAC 주소 사용. (장비: 스위치, 브리지)
  3. **네트워크 계층**: 경로 설정(Routing), 패킷 전달, IP 주소 사용. (장비: 라우터, L3 스위치)
  4. **전송 계층**: 종단 간(End-to-End) 신뢰성 있는 데이터 전송 보장. (TCP, UDP)
  5. **세션 계층**: 프로세스 간의 연결 확립, 유지, 동기화.
  6. **표현 계층**: 데이터 형식 변환, 압축, 암호화.
  7. **응용 계층**: 사용자 인터페이스, 이메일, 파일 전송 등 응용 서비스 제공. (HTTP, FTP, SMTP)


### TCP/IP 프로토콜
- **TCP (Transmission Control Protocol)**: 연결 지향형, 신뢰성 보장, 흐름/혼잡 제어 수행, 순서 제어. (3-Way Handshake로 연결 설정)
- **UDP (User Datagram Protocol)**: 비연결 지향형, 신뢰성 없음, 실시간 전송에 유리(빠름). (오버헤드가 적음)
- **IPv4 vs IPv6**:
  - IPv4: 32비트 (8비트씩 4부분), 유니캐스트, 멀티캐스트, 브로드캐스트.
  - IPv6: 128비트 (16비트씩 8부분), 보안성(IPsec 내장) 및 QoS(품질 보장) 향상, 유니캐스트, 멀티캐스트, 애니캐스트.

### 라우팅 (Routing) 프로토콜
- **RIP (Routing Information Protocol)**: 거리 벡터 알고리즘. 최대 홉(Hop) 수를 15로 제한. 소규모 네트워크에 적합.
- **OSPF (Open Shortest Path First)**: 링크 상태 알고리즘 (다익스트라). 규모가 크고 복잡한 네트워크에 적합하며 변화된 정보만 멀티캐스트로 전송.
- **BGP (Border Gateway Protocol)**: AS(자율 시스템) 간의 라우팅 프로토콜. 경로 벡터 알고리즘 사용.

## 4. 프로그래밍 언어 특징
- **C 언어**: 절차지향, 이식성이 좋음. 하드웨어 직접 제어(포인터 사용).
  - **포인터 (Pointer)**: 메모리의 주소값을 저장하는 변수. (배열과 밀접한 연관)
- **Java 언어**: 객체지향, 플랫폼 독립적(JVM 사용), 가비지 컬렉터(Garbage Collector)를 통한 자동 메모리 관리, 다중 상속 미지원(인터페이스로 대체).
- **Python 언어**: 인터프리터 언어, 객체지향, 동적 타이핑(변수 선언 시 타입 미지정), 들여쓰기(Indent)로 블록 구분, 다양한 라이브러리 지원.

## 5. 정보 보안 (Information Security)
- **보안의 3대 요소 (CIA)**:
  - **기밀성 (Confidentiality)**: 인가된 사용자만 정보에 접근 가능.
  - **무결성 (Integrity)**: 인가된 사용자만 정보를 수정 가능 (정보의 정확성 유지).
  - **가용성 (Availability)**: 인가된 사용자는 언제든지 정보를 사용 가능.

### 암호화 (Encryption) 알고리즘
- **양방향 암호화**:
  - **대칭키 (비밀키) 암호화**: 암호화키와 복호화키가 동일. 속도가 빠르지만 키 분배 관리가 어렵다. (DES, AES, SEED, ARIA)
  - **비대칭키 (공개키) 암호화**: 암호화키(공개)와 복호화키(개인)가 다름. 속도가 느리지만 키 분배가 안전하다. (RSA, ECC)
- **단방향 암호화 (해시, Hash)**: 복호화가 불가능한 암호화. 무결성 검증 및 패스워드 저장에 사용. (MD5, SHA, MAC)

### 주요 사이버 공격 기법
- **DoS / DDoS**: 대량의 데이터를 한 곳의 서버에 집중적으로 전송하여 서버의 정상적인 기능을 방해. (SYN Flooding, Ping of Death, Smurfing)
- **스니핑 (Sniffing)**: 네트워크 상에서 전송되는 데이터를 몰래 엿보는 행위 (수동적 공격).
- **스푸핑 (Spoofing)**: 송신자의 IP, MAC 주소 등을 속여 합법적인 사용자인 것처럼 위장.
- **SQL 인젝션 (SQL Injection)**: 웹 애플리케이션의 입력 폼에 악의적인 SQL 문을 삽입하여 데이터베이스를 조작.
- **XSS (Cross Site Scripting)**: 웹 페이지에 악의적인 스크립트를 삽입하여 사용자의 정보를 탈취.
