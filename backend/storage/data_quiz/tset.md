# 과목:응용_SW_기초_기술_활용 # 키워드: 기억장치 배치 전략, First Fit

문제 : 기억 공간이 15k, 23k, 22k, 21k 순으로 빈 공간이 있을 때 기억장치 배치 전략으로 “*First Fit*”을 사용하여 17k의 프로그램을 적재할 경우 단편화의 크기는 얼마인가?
    1) 5k
    2) 6k 
    3) 7k 
    4) 8k

- 해설: *First Fit*은 첫 번째로 들어갈 수 있는 공간을 선택함. 15k(불가) → **23k(가능)**. 따라서 23k - 17k = 6k의 내부 단편화 발생.
- 정답: **2) 6k**

# 과목:응용_SW_기초_기술_활용 # 키워드: 기억장치 배치 전략, Best Fit

문제: 빈 공간의 크기가 20kb, 16kb, 8kb, 40kb 일 때 기억장치 배치 전략으로 “*Best Fit*”를 사용하여 17kb의 프로그램을 적재할 경우 내부 단편화의 크기는 얼마일까?
    1) 3kb  
    2) 23kb 
    3) 64kb 
    4) 67kb

- 해설: *Best Fit*은 단편화를 가장 작게 남기는 공간을 선택함. 17kb가 들어갈 수 있는 공간(20, 40) 중 가장 적합한 곳은 **20kb**. 따라서 20kb - 17kb = 3kb 발생.
- 정답: **1) 3kb**

문제 : *페이징(Paging)* 기법에서 페이지 크기가 작아질수록 발생하는 현상이 아닌 것은?
    1) 기억장소 이용 효율이 증가한다.  
    2) 입/출력 시간이 늘어난다.  
    3) 내부 단편화가 감소한다.  
    4) 페이지 맵 테이블의 크기가 감소한다.

- 해설: 페이지 크기가 작아지면 관리해야 할 페이지 수가 많아지므로 *페이지 맵 테이블*의 크기는 **증가**함.
