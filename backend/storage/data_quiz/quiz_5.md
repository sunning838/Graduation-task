# 과목: 요구사항 확인 # 키워드: UML, 순차다이어그램, 동적모델링

문제 : UML 다이어그램 중 순차 다이어그램에 대한 설명으로 틀린 것은?

    1) 객체 간의 동적 상호작용을 시간 개념을 중심으로 모델링 하는 것이다.
    2) 주로 시스템의 정적 측면을 모델링하기 위해 사용한다.
    3) 일반적으로 다이어그램의 수직 방향이 시간의 흐름을 나타낸다.
    4) 회귀 메시지(Self-Message), 제어블록(Statement block) 등으로 구성된다.

- 해설: **순차 다이어그램(Sequence Diagram)**은 객체 간에 메시지를 주고받는 상호작용을 시간의 흐름에 따라 나타내는 대표적인 **'동적(Dynamic)'** 모델링 다이어그램입니다. 반면, 시스템의 '정적(Static)' 구조를 모델링할 때 주로 사용하는 것은 클래스(Class) 다이어그램입니다.
- 정답: **2) 주로 시스템의 정적 측면을 모델링하기 위해 사용한다.**

# 과목: 인터페이스 구현 # 키워드: 미들웨어, MOM, 비동기통신

문제 : 메시지 지향 미들웨어(Message-Oriented Middleware, MOM)에 대한 설명으로 틀린 것은?

    1) 느리고 안정적인 응답보다는 즉각적인 응답이 필요한 온라인 업무에 적합하다.
    2) 독립적인 애플리케이션을 하나의 통합된 시스템으로 묶기 위한 역할을 한다.
    3) 송신측과 수신측의 연결 시 메시지 큐를 활용하는 방법이 있다.
    4) 상이한 애플리케이션 간 통신을 비동기 방식으로 지원한다.

- 해설: **메시지 지향 미들웨어(MOM)**는 메시지 큐(Queue)를 통해 데이터를 **'비동기'** 방식으로 주고받습니다. 따라서 송수신이 즉각적으로 이루어지지 않더라도 데이터를 잃어버리지 않고 안정적으로 전달해야 하는 시스템에 적합합니다. 즉각적인 동기식 응답이 필요한 실시간 온라인 업무에는 MOM보다는 TP 모니터(Transaction Processing Monitor)가 훨씬 적합합니다.
- 정답: **1) 느리고 안정적인 응답보다는 즉각적인 응답이 필요한 온라인 업무에 적합하다.**

- 정답: **2) '차량 대여 시스템이 제공하는 모든 화면이 3초 이내에 사용자에게 보여야 한다'는 비기능적 요구이다.**

# 과목: 응용 SW 기초 기술 활용 # 키워드: 객체지향, 다형성, 오버로딩, 오버라이딩

문제 : 객체지향 개념에서 다형성(Polymorphism)과 관련한 설명으로 틀린 것은?

    1) 다형성은 현재 코드를 변경하지 않고 새로운 클래스를 쉽게 추가할 수 있게 한다.
    2) 다형성이란 여러 가지 형태를 가지고 있다는 의미로, 여러 형태를 받아들일 수 있는 특징을 말한다.
    3) 메소드 오버라이딩(Overriding)은 상위 클래스에서 정의한 일반 메소드의 구현을 하위 클래스에서 무시하고 재정의 할 수 있다.
    4) 메소드 오버로딩(Overloading)의 경우 매개 변수 타입은 동일하지만 메소드명을 다르게 함으로써 구현, 구분할 수 있다.

- 해설: 객체지향의 다형성을 구현하는 두 가지 핵심 기법은 오버라이딩과 오버로딩입니다.
    * **오버라이딩(Overriding)**: 상속 관계에서 부모 클래스의 메서드를 자식 클래스에서 재정의(덮어쓰기)하는 것입니다. (메서드명, 매개변수 모두 동일)
    * **오버로딩(Overloading)**: 같은 클래스 내에서 **메서드 이름은 '동일'**하게 하되, **매개변수의 '타입이나 개수'를 다르게** 하여 여러 개를 정의하는 것입니다. 4번 보기는 거꾸로 설명하고 있으므로 틀렸습니다.

- 정답: **4) 메소드 오버로딩(Overloading)의 경우 매개 변수 타입은 동일하지만 메소드명을 다르게 함으로써 구현, 구분할 수 있다.**

# 과목: 통합 구현 # 키워드: 통합테스트, 하향식통합, 상향식통합

    2) 하향식 통합 테스트의 경우 넓이 우선(Breadth First) 방식으로 테스트를 할 모듈을 선택할 수 있다.
    3) 상향식 통합 테스트의 경우 시스템 구조도의 최상위에 있는 모듈을 먼저 구현하고 테스트한다.
    4) 모듈 간의 인터페이스와 시스템의 동작이 정상적으로 잘 되고 있는지를 빨리 파악하고자 할 때 상향식 보다는 하향식 통합 테스트를 사용하는 것이 좋다.

- 해설: 통합 테스트 방식 중 **상향식(Bottom-up) 통합 테스트**는 시스템의 가장 하위 계층에 있는 모듈부터 순차적으로 구현하고 테스트하며 위로 올라가는 방식입니다. 반대로 시스템의 최상위 모듈부터 아래로 내려가며 테스트하는 방식은 **하향식(Top-down)**입니다.
    * 하향식 통합 테스트: 상위 모듈에서 하위 모듈 방향으로 통합. 하위 모듈의 역할을 대신하는 **스텁(Stub)**이 필요합니다.
    * 상향식 통합 테스트: 하위 모듈에서 상위 모듈 방향으로 통합. 상위 모듈의 역할을 대신하는 **드라이버(Driver)**가 필요합니다.
- 정답: **3) 상향식 통합 테스트의 경우 시스템 구조도의 최상위에 있는 모듈을 먼저 구현하고 테스트한다.**

    이처럼 탐색 범위를 반씩 잘라가며 총 3번의 비교 연산을 수행하게 됩니다.

# 과목: 데이터 입출력 구현 # 키워드: 데이터베이스, 정규화, BCNF

문제 : 다음 조건을 모두 만족하는 정규형은?
> 
- 테이블 R에 속한 모든 도메인이 원자값만으로 구성되어 있다.
- 테이블 R에서 키가 아닌 모든 필드가 키에 대해 함수적으로 종속되며, 키의 부분 집합이 결정자가 되는 부분 종속이 존재하지 않는다.
- 테이블 R에 존재하는 모든 함수적 종속에서 결정자가 후보키이다.

    1) BCNF
    2) 제1정규형
    3) 제2정규형
    4) 제3정규형

    * 첫 번째 조건(모든 도메인이 원자값)은 **제1정규형(1NF)**을 의미합니다.
    * 두 번째 조건(부분 함수 종속 제거)은 **제2정규형(2NF)**을 의미합니다.
    * 세 번째 조건(모든 결정자가 후보키)은 **BCNF(Boyce-Codd Normal Form)**의 핵심적인 정의입니다.
    제시된 모든 조건을 누적하여 충족한 상태를 뜻하므로 가장 강력한 조건인 BCNF가 정답이 됩니다. (참고로 제3정규형은 기본키가 아닌 일반 속성들끼리의 이행적 함수 종속을 제거한 상태를 말합니다.)
- 정답: **1) BCNF**

# 과목: 데이터 입출력 구현 # 키워드: 데이터베이스, 트랜잭션, ACID, 원자성

문제 : 데이터베이스의 트랜잭션 성질들 중에서 다음 설명에 해당하는 것은?
> 
트랜잭션의 모든 연산들이 정상적으로 수행 완료되거나 아니면 전혀 어떠한 연산도 수행되지 않은 원래 상태가 되도록 해야 한다.

    1) Atomicity
    2) Consistency
    3) Isolation
    4) Durability

- 해설: 데이터베이스 트랜잭션이 안전하게 수행되기 위한 4가지 주요 특성(ACID) 중 **원자성(Atomicity)**에 대한 설명입니다. 작업이 완벽하게 끝마쳐져서 데이터베이스에 반영되거나(Commit), 중간에 오류가 발생하면 아예 실행되지 않은 초기 상태로 완벽히 되돌려져야(Rollback) 한다는 'All or Nothing'의 원칙을 의미합니다.
    * Consistency(일관성): 트랜잭션 수행 전후에 데이터베이스가 모순 없이 일관된 상태를 유지해야 합니다.
    * Isolation(독립성/격리성): 실행 중인 트랜잭션 중간에 다른 트랜잭션이 끼어들어 연산 결과를 훼손할 수 없습니다.
    * Durability(영속성/지속성): 성공적으로 완료(Commit)된 트랜잭션의 결과는 시스템이 고장 나더라도 영구적으로 보존되어야 합니다.
- 정답: **1) Atomicity**

# 과목: 데이터 입출력 구현 # 키워드: 데이터베이스, 분산데이터베이스, 분산투명성

문제 : 분산 데이터베이스 시스템과 관련한 설명으로 틀린 것은?

    1) 물리적으로 분산된 데이터베이스 시스템을 논리적으로 하나의 데이터베이스 시스템처럼 사용할 수 있도록 한 것이다.
    2) 물리적으로 분산되어 지역별로 필요한 데이터를 처리할 수 있는 지역 컴퓨터(Local Computer)를 분산 처리기 (Distributed Processor)라고 한다.
    3) 분산 데이터베이스 시스템을 위한 통신 네트워크 구조가 데이터 통신에 영향을 주므로 효율적으로 설계해야 한다.
    4) 데이터베이스가 분산되어 있음을 사용자가 인식할 수 있도록 분산 투명성(Distribution Transparency)을 배제해야 한다.

- 해설: **분산 데이터베이스 시스템**의 핵심 목표는 사용자가 데이터베이스의 물리적인 위치나 분산 상태를 전혀 신경 쓰지 않고, 마치 자신의 PC에 있는 하나의 데이터베이스를 사용하는 것처럼 느끼게 하는 것입니다. 이를 **분산 투명성(Distribution Transparency)**이라고 하며, 배제하는 것이 아니라 반드시 보장하고 완벽하게 제공해야 하는 필수 속성입니다.
- 정답: **4) 데이터베이스가 분산되어 있음을 사용자가 인식할 수 있도록 분산 투명성(Distribution Transparency)을 배제해야 한다.**

# 과목: 데이터 입출력 구현 # 키워드: 관계형데이터베이스, 릴레이션특징

    * **Best Effort**: 전송의 신뢰성을 보장하지 않고 최선을 다해 전달만 시도합니다.
    * **비연결형**: 사전 연결 설정 없이 패킷을 독립적으로 전송합니다.
- 정답: **1) 체크섬(Checksum) 기능으로 데이터 체크섬(Data Checksum)만 제공한다.**

# 과목: 응용 SW 기초 기술 활용 # 키워드: 모듈, 결합도, Coupling

    1) 내용 결합도(Content Coupling)
    2) 제어 결합도(Control Coupling)
    3) 공통 결합도(Common Coupling)
    4) 스탬프 결합도(Stamp Coupling)

- 해설: **결합도(Coupling)**는 모듈과 모듈 사이의 상호 의존 정도를 나타냅니다. 결합도는 **낮을수록(약할수록)** 좋은 설계이며, 가장 높은 단계(나쁜 설계)부터 낮은 단계 순서는 다음과 같습니다.
    * **내용(Content)** > 공통(Common) > 외부(External) > 제어(Control) > 스탬프(Stamp) > **자료(Data)**
    문제에서처럼 한 모듈이 다른 모듈의 내부를 직접 참조하거나 수정하는 경우를 **내용 결합도**라고 하며, 이는 독립성을 완전히 해치는 가장 나쁜 결합도 단계입니다.
- 정답: **1) 내용 결합도(Content Coupling)**

# 과목: 응용 SW 기초 기술 활용 # 키워드: C언어, 배열주소, 메모리구조

문제 : a[0]의 주소값이 10일 경우 다음 C언어 프로그램이 실행되었을 때의 결과는? (단, int 형의 크기는 4Byte로 가정한다.)
> 
#include <stdio.h>
int main(int argc, char *argv[]) {
    int a[] = {14, 22, 30, 38} ;
    printf("%u, ", &a[2]) ;
    printf("%u", a) ;
    return 0 ;
}

    1) 14, 10
    2) 14, 14
    3) 18, 10
    4) 18, 14

- 해설: C언어의 배열과 메모리 주소 계산 방식을 이해해야 합니다.
    * **주소 계산**: `int`형은 4바이트이므로, 배열 `a`의 각 요소 주소는 4씩 증가합니다.
        - `a[0]`의 주소: 10
        - `a[1]`의 주소: $10 + 4 = 14$
        - `a[2]`의 주소: $10 + (2 \times 4) = 18$
    * **출력 분석**:
        - `&a[2]`: 배열의 3번째 요소인 `a[2]`의 **주소값**을 출력하라는 의미이므로 **18**입니다.
        - `a`: 배열의 이름은 그 자체로 **배열의 시작 주소**(`&a[0]`)를 의미하므로 **10**입니다.
- 정답: **3) 18, 10**

    3) Authorization
    4) Authentication

- 해설: 시스템 보안에서 사용자의 신원을 확인하는 과정을 **인증(Authentication)**이라고 합니다. 아이디와 비밀번호를 입력하거나 생체 정보를 인식하는 것이 대표적인 예입니다. 반면, 인증된 사용자가 특정 데이터나 기능에 접근할 수 있는 권한이 있는지 확인하는 과정은 **인가(Authorization)**라고 하니 두 개념을 꼭 구분해서 기억해 두세요.
- 정답: **4) Authentication**

# 과목: 응용 SW 기초 기술 활용 # 키워드: 네트워크, 신기술, SDN

문제 : 다음에서 설명하는 IT 기술은?
> 
- 네트워크를 제어부, 데이터 전달부로 분리하여 네트워크 관리자가 보다 효율적으로 네트워크를 제어, 관리할 수 있는 기술
- 기존의 라우터, 스위치 등과 같이 하드웨어에 의존하는 네트워크 체계에서 안정성, 속도, 보안 등을 소프트웨어로 제어, 관리하기 위해 개발됨
- 네트워크 장비의 펌웨어 업그레이드를 통해 사용자의 직접적인 데이터 전송 경로 관리가 가능하고, 기존 네트워크에는 영향을 주지 않으면서 특정 서비스의 전송 경로 수정을 통하여 인터넷상에서 발생하는 문제를 처리할 수 있음

    1) SDN(Software Defined Networking)
    2) NFS(Network File System)
    3) Network Mapper
    4) AOE Network

- 해설: **SDN(소프트웨어 정의 네트워킹)**은 네트워크의 제어 부분(Control Plane)과 데이터 전달 부분(Data Plane)을 분리하여, 네트워크 인프라를 소프트웨어로 프로그래밍하고 중앙에서 통합 제어할 수 있게 해주는 차세대 네트워크 기술입니다. 고가의 하드웨어 장비에 종속되지 않고 유연하게 네트워크 경로와 보안을 관리할 수 있다는 것이 핵심입니다.
- 정답: **1) SDN(Software Defined Networking)**

# 과목: 요구사항 확인 # 키워드: 프로젝트관리, 일정관리, PERT차트

문제 : 프로젝트 일정 관리 시 사용하는 PERT 차트에 대한 설명에 해당하는 것은?

    1) 각 작업들이 언제 시작하고 언제 종료되는지에 대한 일정을 막대 도표를 이용하여 표시한다.
    2) 시간선(Time-line) 차트라고도 한다.
    3) 수평 막대의 길이는 각 작업의 기간을 나타낸다.
    4) 작업들 간의 상호 관련성, 결정경로, 경계시간, 자원할당 등을 제시한다.

- 해설: 본문(박스)이 없는 문제이므로 지시해주신 대로 인용 기호 없이 깔끔하게 정리했습니다. 
프로젝트 일정 관리에 사용되는 도구 중 1, 2, 3번 보기는 수평 막대 형태인 **간트 차트(Gantt Chart)**에 대한 설명입니다. 반면 **PERT(Program Evaluation and Review Technique) 차트**는 작업들 간의 선후 관계(상호 관련성), 임계 경로(결정 경로), 소요 시간 등을 동그라미(노드)와 화살표(간선)로 연결된 네트워크 그래프 형태로 제시하는 도구입니다.
- 정답: **4) 작업들 간의 상호 관련성, 결정경로, 경계시간, 자원할당 등을 제시한다.**
