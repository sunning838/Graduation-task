# 과목: 요구사항 확인 # 키워드: SOLID, 클래스설계원칙, 개방폐쇄원칙

문제 : 클래스 설계원칙에 대한 바른 설명은?

    1) 단일 책임원칙 : 하나의 클래스만 변경 가능 해야한다.
    2) 개방-폐쇄의 원칙 : 클래스는 확장에 대해 열려 있어야 하며 변경에 대해 닫혀 있어야 한다.
    3) 리스코프 교체의 원칙 : 여러 개의 책임을 가진 클래스 는 하나의 책임을 가진 클래스로 대체되어야 한다.
    4) 의존관계 역전의 원칙 : 클라이언트는 자신이 사용하는 메소드와 의존관계를 갖지 않도록 해야 한다.

    * 단일 책임 원칙(SRP): 하나의 클래스는 하나의 책임(이유)만 가져야 함.
    * 리스코프 교체 원칙(LSP): 자식 클래스는 언제나 부모 클래스를 대체할 수 있어야 함.
- 정답: **2) 개방-폐쇄의 원칙 : 클래스는 확장에 대해 열려 있어야 하며 변경에 대해 닫혀 있어야 한다.**

ㄴ. 순서 리스트의 뒤(Rear)에서 노드가 삽입되며, 앞(Front)에서 노드가 제거된다.

    * ㄷ: 양쪽 끝에서 삽입과 삭제가 모두 가능한 구조는 **데크(Deque)**입니다.

# 과목: 응용 SW 기초 기술 활용 # 키워드: C언어, 2차원배열, 배열포인터, 포인터연산

문제 : 다음 C언어 프로그램이 실행되었을 때, 실행 결과는?
> 
#include <stdio.h>
#include <stdlib.h>
int main(int argc, char *argv[]) {
    int arr[2][3]={1,2,3,4,5,6};
    int (*p)[3]=NULL;
    p=arr;
    printf("%d, ", *(p[0]+1) + *(p[1]+2));
    printf("%d", *(*(p+1)+0) + *(*(p+1)+1));
    return 0;
}

    1) 7, 5
    2) 8, 5
    3) 8, 9
    4) 7, 9

- 해설: 2차원 배열과 배열 포인터를 활용한 C언어의 포인터 연산 문제입니다.
    * 2차원 배열 `arr[2][3]`은 메모리에 순차적으로 저장되며 논리적으로 다음과 같습니다.
      `arr[0]` 행: `{1, 2, 3}` (인덱스: `arr[0][0]`, `arr[0][1]`, `arr[0][2]`)
      `arr[1]` 행: `{4, 5, 6}` (인덱스: `arr[1][0]`, `arr[1][1]`, `arr[1][2]`)
    * `int (*p)[3]=NULL;`은 크기가 3인 1차원 정수 배열을 가리키는 **배열 포인터**입니다. `p=arr;`를 통해 `p`는 `arr`의 첫 번째 행(`arr[0]`)을 가리키게 됩니다.
    * **첫 번째 printf 연산: `*(p[0]+1) + *(p[1]+2)`**
      - `p[0]`은 `arr[0]`과 동일하므로 1행의 시작 주소를 의미합니다. `p[0]+1`은 `arr[0][1]`의 주소이며, 이를 역참조(`*`)한 값은 **2**입니다.
      - `p[1]`은 `arr[1]`과 동일하므로 2행의 시작 주소를 의미합니다. `p[1]+2`는 `arr[1][2]`의 주소이며, 이를 역참조(`*`)한 값은 **6**입니다.
      - 2 + 6 = **8**이 먼저 출력됩니다.
    * **두 번째 printf 연산: `*(*(p+1)+0) + *(*(p+1)+1)`**
      - `(p+1)`은 포인터 연산에 의해 다음 행인 두 번째 행(`arr[1]`) 전체를 가리킵니다. 이를 한 번 역참조한 `*(p+1)`은 2행의 시작 주소(`&arr[1][0]`)로 변환됩니다.
      - `*(p+1)+0`은 `arr[1][0]`의 주소이며, 다시 역참조(`*`)하면 값은 **4**입니다.
      - `*(p+1)+1`은 `arr[1][1]`의 주소이며, 다시 역참조(`*`)하면 값은 **5**입니다.
      - 4 + 5 = **9**가 뒤이어 출력됩니다.
- 정답: **3) 8, 9**

# 과목: 응용 SW 기초 기술 활용 # 키워드: JAVA, 배열, 반복문, 조건문

문제 : 다음 JAVA 프로그램이 실행되었을 때, 실행 결과는?
> 
public class Ape{
    static void rs(char a[]) {
        for(int i = 0; i < a.length; i++)
            if(a[i] == 'B')
                a[i] = 'C';
            else if(i == a.length - 1)
                a[i] = a[i-1];
            else a[i] = a[i+1];
    }

    static void pca(char a[]) {
        for(int i = 0; i < a.length; i++)
            System.out.print(a[i]);
        System.out.println();
    }

    public static void main(String[] args) {
        char c[] = {'A', 'B', 'D', 'D', 'A', 'B', 'C'};
        rs(c);
        pca(c);
    }
}

    1) BCDABCA
    2) BCDABCC
    3) CDDACCC
    4) CDDACCA

- 해설: 초기 배열 `c`의 상태는 `{'A', 'B', 'D', 'D', 'A', 'B', 'C'}`이며, 배열의 길이는 7입니다. `rs()` 메서드 내의 반복문을 돌며 배열의 값이 실시간으로 변경(In-place)되는 과정을 추적해야 합니다.

>

    1) 0
    2) 1
    3) 3
    4) 5

- 해설: C언어의 연산자 우선순위를 정확히 알아야 하는 문제입니다. 이 식은 **산술 연산 > 시프트 연산 > 관계 연산 > 논리 연산** 순서로 처리됩니다.
    * 변수 대입: `1 < 2 + 2 && 1 << 1 <= 2`

# 과목: 소프트웨어 개발 보안 구축  # 키워드: 스위치재밍, MAC플러딩, 네트워크보안

    2) LAN Tapping
    3) Switch Jamming
    4) FTP Flooding

- 해설: 스위치(Switch)는 목적지 MAC 주소를 기억해 해당 포트로만 데이터를 은밀하게 보내는 지능형 장비입니다. 하지만 내부에 MAC 주소를 저장하는 메모리(테이블) 용량에는 한계가 있습니다. 공격자가 이 점을 노려 위조된 수많은 MAC 주소를 계속 전송하면 테이블이 꽉 차버리게 되고(오버플로우), 스위치는 결국 정상적인 작동을 포기한 채 들어오는 모든 데이터를 모든 포트로 무작정 뿌려버리는 '더미 허브(Dummy Hub)'처럼 동작하게 됩니다. 이를 통해 공격자는 자신에게 오지 말아야 할 패킷들까지 전부 훔쳐볼 수(스니핑) 있게 되며, 이 기법을 **스위치 재밍(Switch Jamming)** 또는 MAC 플러딩(MAC Flooding)이라고 부릅니다.

# 과목: 소프트웨어 개발 보안 구축 # 키워드: 네트워크보안, DoS공격, Smurf

문제 : DoS(Denial of Service) 공격과 관련한 내용으로 틀린 것은?

    1) Ping of Death 공격은 정상 크기보다 큰 ICMP 패킷을 작은 조각(Fragment)으로 쪼개어 공격 대상이 조각화 된 패킷을 처리하게 만드는 공격 방법이다.
    2) Smurf 공격은 멀티캐스트(Multicast)를 활용하여 공격 대상이 네트워크의 임의의 시스템에 패킷을 보내게 만드는 공격이다.
    3) SYN Flooding은 존재하지 않는 클라이언트가 서버별로 한정된 접속 가능 공간에 접속한 것처럼 속여 다른 사용자가 서비스를 이용하지 못하게 하는 것이다.
    4) Land 공격은 패킷 전송 시 출발지 IP주소와 목적지 IP주소 값을 똑같이 만들어서 공격 대상에게 보내는 공격 방법이다.

- 해설: **Smurf(스머프) 공격**은 출발지 IP를 공격 대상의 IP로 위조한 뒤, 특정 네트워크 내의 '모든' 시스템에게 ICMP 요청(Ping)을 보냅니다. 이때 특정 그룹에게만 보내는 멀티캐스트(Multicast)가 아니라, 네트워크의 모든 호스트에게 메시지를 보내는 **브로드캐스트(Broadcast)**를 활용합니다. 브로드캐스트를 받은 수많은 호스트들은 일제히 위조된 IP(공격 대상)로 응답 패킷을 보내게 되어, 타겟 서버가 트래픽 폭주로 마비됩니다.
- 정답: **2) Smurf 공격은 멀티캐스트(Multicast)를 활용하여 공격 대상이 네트워크의 임의의 시스템에 패킷을 보내게 만드는 공격이다.**

# 과목: 응용 SW 기초 기술 활용 # 키워드: 리눅스, 파일권한, umask

문제 : 리눅스에서 생성된 파일 권한이 644일 경우 umask 값은?

    1) 022
    2) 666
    3) 777
    4) 755

- 해설: 리눅스에서 파일 생성 시 기본 최대 권한은 **666** (디렉터리는 777)입니다. 실제 생성되는 파일의 권한은 `최대 권한 - umask`로 계산됩니다.
    * 666(기본) - umask = 644(결과)
    * 따라서 umask = 666 - 644 = **022**가 됩니다.
- 정답: **1) 022**

# 과목: 응용 SW 기초 기술 활용 # 키워드: 프로젝트관리, 비용산정, LOC

문제 : 상향식 비용 산정 기법 중 LOC(원시 코드 라인 수) 기법에서 예측치를 구하기 위해 사용하는 항목이 아닌 것은?

    1) 낙관치
    2) 기대치
    3) 비관치
    4) 모형치

- 해설: LOC 기법으로 산정할 때, 예측값($E$)은 **낙관치**(가장 적게 들 때), **비관치**(가장 많이 들 때), **기대치**(보통일 때, 중간치)를 활용하여 계산합니다. 계산식은 다음과 같습니다.
$$E = \frac{낙관치 + 4 \times 기대치 + 비관치}{6}$$
    * 모형치라는 항목은 사용하지 않습니다.
- 정답: **4) 모형치**

# 과목: 소프트웨어 개발 보안 구축 # 키워드: 패킷분석, DPI, 네트워크보안

문제 : OSI 7 Layer 전 계층의 프로토콜과 패킷 내부의 콘텐츠를 파악하여 침입 시도, 해킹 등을 탐지하고 트래픽을 조정하기 위한 패킷 분석 기술은?

    1) PLCP(Packet Level Control Processor)
    2) Traffic Distributor
    3) Packet Tree
    4) DPI(Deep Packet Inspection)

- 해설: **DPI(Deep Packet Inspection)**는 일반적인 패킷 분석(포트나 IP 중심)보다 훨씬 깊게, 패킷의 데이터 본문(Payload) 내용까지 샅샅이 검사하여 해킹 여부나 특정 애플리케이션 여부를 판단하는 고급 분석 기술입니다.
- 정답: **4) DPI(Deep Packet Inspection)**

# 과목: 요구사항 확인 # 키워드: 개발방법론, 테일러링, Tailoring

문제 : 소프트웨어 개발 방법론의 테일러링(Tailoring)과 관련한 설명으로 틀린 것은?

    1) 프로젝트 수행 시 예상되는 변화를 배제하고 신속히 진행하여야 한다.
    2) 프로젝트에 최적화된 개발 방법론을 적용하기 위해 절차, 산출물 등을 적절히 변경하는 활동이다.
    3) 관리 측면에서의 목적 중 하나는 최단기간에 안정적인 프로젝트 진행을 위한 사전 위험을 식별하고 제거하는 것이다.
    4) 기술적 측면에서의 목적 중 하나는 프로젝트에 최적화된 기술 요소를 도입하여 프로젝트 특성에 맞는 최적의 기법과 도구를 사용하는 것이다.

- 해설: **테일러링(Tailoring)**은 표준 방법론을 기반으로 하되, 각 프로젝트의 규모, 성격, **환경 변화** 등에 맞춰 절차나 도구를 유연하게 조정(맞춤 제작)하는 기법입니다. 따라서 예상되는 변화를 배제하는 것이 아니라, 오히려 그러한 특성과 변화를 적극적으로 반영하여 방법론을 최적화하는 것이 목적입니다.
- 정답: **1) 프로젝트 수행 시 예상되는 변화를 배제하고 신속히 진행하여야 한다.**
