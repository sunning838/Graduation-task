# 과목: 소프트웨어_설계 # 키워드: 시스템_구성요소, Process, Feedback, Control

문제 1. 시스템의 구성요소로 볼 수 없는 것은?

1) Process
2) Feedback
3) Maintenance
4) Control

정답: 3
해설: 시스템이 본래의 목적을 달성하기 위해 갖추어야 할 5대 기본 구성요소는 입력(Input), 처리(Process), 출력(Output), 제어(Control), 피드백(Feedback)입니다. 유지보수(Maintenance)는 소프트웨어 개발 생명주기(SDLC)의 한 과정일 뿐, 시스템 자체를 구성하는 5대 기본 요소에는 포함되지 않습니다.

문제 2. 유스케이스(Usecase)에 대한 설명 중 옳은 것은?

1) 유스케이스 다이어그램은 개발자의 요구를 추출하고 분석하기 위해 주로 사용한다.
2) 액터는 대상 시스템과 상호 작용하는 사람이나 다른 시스템에 의한 역할이다.
3) 사용자 액터는 본 시스템과 데이터를 주고받는 연동 시스템을 의미한다.
4) 연동의 개념은 일방적으로 데이터를 파일이나 정해진 형식으로 넘겨주는 것을 의미한다.

정답: 2
해설: 액터(Actor)는 시스템 외부에 존재하면서 시스템과 상호작용하는 모든 객체(사람, 기기, 외부 시스템 등)를 의미하며, 이는 2번의 설명과 정확히 일치합니다.
[오답 분석] 
1번: 유스케이스는 '개발자'가 아닌 '사용자(고객)' 관점에서의 요구사항을 추출하고 분석하기 위해 사용됩니다.
3번: 본 시스템과 데이터를 주고받는 연동 시스템은 '사용자 액터'가 아니라 '시스템 액터'에 해당합니다. (사용자 액터는 실제 사람을 의미함)
4번: 연동은 일방적인 데이터 전달뿐만 아니라 양방향 통신 및 상호작용을 포괄하는 개념이며, 이는 유스케이스 자체에 대한 올바른 설명과 거리가 멉니다.

# 과목: 소프트웨어_설계 # 키워드: 요구사항_개발, 도출, 분석, 명세, 확인

문제 3. 요구사항 개발 프로세스의 순서로 옳은 것은?

㉠ 도출(Elicitation)    ㉡ 분석(Analysis)
㉢ 명세(Specification)  ㉣ 확인(Validation)

1) ㉠ - ㉡ - ㉢ - ㉣
2) ㉠ - ㉢ - ㉡ - ㉣
3) ㉠ - ㉣ - ㉡ - ㉢
4) ㉠ - ㉡ - ㉣ - ㉢

정답: 1
해설: 요구사항 개발 프로세스는 타당성 조사 이후 [도출 -> 분석 -> 명세 -> 확인]의 4단계로 진행됩니다. 
1. 도출(Elicitation): 요구사항이 어디에 있는지, 어떻게 수집할 것인지 식별 (인터뷰, 설문 등)
2. 분석(Analysis): 도출된 요구사항의 충돌, 중복 등을 해결하고 개념 모델링 수행
3. 명세(Specification): 분석된 요구사항을 바탕으로 체계적인 문서 작성 (SRS 등)
4. 확인(Validation): 명세서가 사용자의 요구를 정확히 반영했는지 검증 및 리뷰

문제 4. 객체지향 기법에서 같은 클래스에 속한 각각의 객체를 의미하는 것은?

1) instance
2) message
3) method
4) module

정답: 1
해설: 객체지향 프로그래밍에서 '인스턴스(instance)'는 특정 클래스(Class)를 바탕으로 메모리에 할당되어 실제 생성된 각각의 구체적인 객체(Object)를 의미합니다.

4번: 모듈(module)은 프로그램을 구성하는 독립적인 논리적 부분 또는 파일 단위로, 클래스보다 더 큰 개념의 구조적 단위입니다.

# 과목: 소프트웨어_설계 # 키워드: 객체지향, 캡슐화, 정보은닉

문제 5. 객체지향 설계에서 객체가 가지고 있는 속성과 오퍼레이션의 일부를 감추어서 객체의 외부에서는 접근이 불가능하게 하는 개념은? (문제 오류로 가답안 발표시 3번으로 발표되었지만 확정 답안 발표시 2, 3번이 정답처리 되었습니다. 여기서는 가답안인 3번을 누르면 정답 처리 됩니다.)

1) 조직화(Organizing)
2) 캡슐화(Encapsulation)
3) 정보은닉(Infomation Hiding)
4) 구조화(Structuralization)

정답: 3
해설: 내부의 상세한 구현을 외부에 감추어 접근을 제한하는 설계 '개념/원칙' 자체는 '정보 은닉(Information Hiding)'입니다. 그리고 이 정보 은닉을 달성하기 위해 관련된 데이터(속성)와 함수(오퍼레이션)를 하나의 클래스라는 캡슐로 묶어 보호하는 '구현 기법'이 '캡슐화(Encapsulation)'입니다. 두 개념이 밀접하게 연결되어 있어 실제 시험에서는 2번과 3번이 모두 복수 정답으로 인정되었으나, 지문의 지시사항에 따라 가답안인 3번을 정답으로 처리합니다. (참고: 원칙을 엄격히 지키기 위해 보기 3번 영문의 'Infomation' 오타도 원문 그대로 기재했습니다.)

# 과목: 소프트웨어_설계 #키워드: GoF, 디자인_패턴, Bridge, Adapter

문제 6. GoF (Gangs of Four) 디자인 패턴에 대한 설명으로 틀린 것은?

1) factory method pattern은 상위클래스에서 객체를 생성하는 인터페이스를 정의하고, 하위클래스에서 인스턴스를 생성하도록 하는 방식이다.
2) prototype pattern은 prototype을 먼저 생성하고 인스턴스를 복제하여 사용하는 구조이다.
3) bridge pattern은 기존에 구현되어 있는 클래스에 기능 발생 시 기존 클래스를 재사용할 수 있도록 중간에서 맞춰주는 역할을 한다.
4) mediator pattern은 객체간의 통제와 지시의 역할을 하는 중재자를 두어 객체지향의 목표를 달성하게 해준다.

정답: 3
해설: 보기 3번의 설명은 '어댑터(Adapter) 패턴'에 해당합니다. 어댑터 패턴은 인터페이스 호환성 문제로 같이 쓸 수 없는 클래스들을 중간에서 연결(변환)해주어 재사용할 수 있게 만드는 패턴입니다. 반면, '브리지(Bridge) 패턴'은 기능의 클래스 계층(추상화)과 구현의 클래스 계층을 분리하여 서로 독립적으로 확장하고 변형할 수 있게 해주는 패턴입니다.
[오답 분석]
1번, 2번은 객체 생성에 관련된 생성(Creational) 패턴에 대한 올바른 설명이며, 4번은 객체들 간의 상호작용을 캡슐화하는 행위(Behavioral) 패턴 중 하나인 중재자 패턴의 올바른 설명입니다.

문제 7. 요구사항 분석이 어려운 이유가 아닌 것은?

1) 개발자와 사용자 간의 지식이나 표현의 차이가 커서 상호 이해가 쉽지 않다.
2) 사용자의 요구는 예외가 거의 없어 열거와 구조화가 어렵지 않다.
3) 사용자의 요구사항이 모호하고 불명확하다.

# 과목: 소프트웨어_설계 # 키워드: 인터페이스_구현, 시스템_인터페이스, 송신_시스템

문제 9. 다음 설명에 해당하는 시스템으로 옳은 것은?

시스템 인터페이스를 구성하는 시스템으로, 연계할 데이터를 데이터베이스와 애플리케이션으로부터 연계 테이블 또는 파일 형태로 생성하여 송신하는 시스템이다.

1) 연계 서버
2) 중계 서버
3) 송신 시스템
4) 수신 시스템

정답: 3
해설: 박스 안의 설명은 **송신 시스템**의 정확한 정의입니다. 시스템 인터페이스 연계 구성 요소는 크게 3가지로 나뉩니다.

* **수신 시스템:** 외부로부터 수신받은 데이터를 변환하여 내부 DB나 애플리케이션에 반영하는 시스템입니다.
* **중계(연계) 서버:** 송신 시스템과 수신 시스템 사이에서 데이터 송수신 현황을 모니터링하고 데이터 형식 변환, 보안 등을 담당하는 서버입니다.

# 과목: 소프트웨어_설계 # 키워드: 소프트웨어_아키텍처, 아키텍처_스타일, 파이프_필터

문제 12. 서브시스템이 입력 데이터를 받아 처리하고 결과를 다른 시스템에 보내는 작업이 반복되는 아키텍처 스타일은?

1) 클라이언트 서버 구조
2) 계층 구조
3) MVC 구조
4) 파이프 필터 구조

정답: 4
해설: **파이프-필터(Pipe-Filter) 구조**는 데이터 흐름 기반의 아키텍처 형태로, 각 서브시스템(필터)이 입력 데이터를 받아 처리한 후 그 결과를 다음 서브시스템으로 넘겨주는(파이프) 과정이 반복되는 스타일입니다. 유닉스(UNIX)의 쉘(Shell) 명령어 파이프라인이 대표적인 예시입니다.
[오답 분석]
1번 클라이언트 서버 구조: 서비스를 요청하는 클라이언트와 요청된 서비스를 제공하는 서버로 분리되는 네트워크 기반 아키텍처입니다.
2번 계층 구조: 시스템을 여러 계층(Layer)으로 나누어 각 계층이 특정 역할을 수행하며, 인접한 계층끼리만 통신하는 구조입니다.
3번 MVC 구조: 시스템을 모델(Model), 뷰(View), 컨트롤러(Controller) 3가지로 나누어 사용자 인터페이스(UI)와 비즈니스 로직을 분리하는 설계 패턴입니다.

문제 13. 럼바우(Rumbaugh)의 객체지향 분석에서 사용하는 분석 활동으로 옳은 것은?

2) 객체 모델링, 동적 모델링, 기능 모델링

정답: 2
해설: 럼바우(Rumbaugh)의 객체지향 분석 기법(OMT: Object Modeling Technique)은 모든 소프트웨어 구성 요소를 그래픽 표기법을 이용하여 모델링하는 기법입니다. 분석 활동은 **[객체 모델링 -> 동적 모델링 -> 기능 모델링]** 순서로 진행됩니다. (암기 팁: '객동기')

* **기능(Functional) 모델링:** 자료 흐름도(DFD)를 이용하여 다수의 프로세스들 간의 자료 흐름을 중심으로 처리 과정을 표현합니다.

# 과목: 소프트웨어_개발 # 키워드: 소프트웨어_테스트, 테스트_원칙, 결함_집중, 파레토_법칙

문제 23. 다음 설명의 소프트웨어 테스트의 기본원칙은?

○ 파레토 법칙이 좌우한다.
○ 애플리케이션 결함의 대부분은 소수의 특정한 모듈에 집중되어 존재한다.
○ 결함은 발생한 모듈에서 계속 추가로 발생할 가능성이 높다.

1) 살충제 패러독스
2) 결함 집중
3) 오류 부재의 궤변
4) 완벽한 테스팅은 불가능

정답: 2
해설: 박스 안의 설명은 소프트웨어 테스트의 7가지 기본 원칙 중 **결함 집중(Defect Clustering)**에 대한 설명입니다. 이는 "80%의 결함은 20%의 특정 모듈에서 발견된다"는 파레토 법칙(Pareto Principle)을 소프트웨어 테스트에 적용한 것입니다.
[오답 분석]
1번 살충제 패러독스(Pesticide Paradox): 동일한 테스트 케이스로 반복 실행하면 더 이상 새로운 결함을 발견할 수 없으므로 테스트 케이스를 주기적으로 리뷰하고 개선해야 한다는 원칙입니다.
3번 오류 부재의 궤변(Absence of Errors Fallacy): 결함을 모두 제거했더라도 사용자의 요구사항을 충족시키지 못한다면 해당 소프트웨어는 품질이 높다고 볼 수 없다는 원칙입니다.
4번 완벽한 테스팅은 불가능(Testing shows presence of defects): 소프트웨어의 모든 상황과 경로를 완벽하게 테스트하는 것은 물리적, 시간적으로 불가능하다는 원칙입니다.

# 과목: 소프트웨어_개발 # 키워드: 자료구조, 스택, 후위표기법, Postfix

문제 33. 다음 postfix로 표현된 연산식의 연산 결과로 옳은 것은?

3 4 * 5 6 * +

1) 35
2) 42
3) 81
4) 360

정답: 2
해설: 후위 표기법(Postfix)은 연산자가 피연산자 뒤에 위치하는 표기법으로, 스택(Stack) 자료구조를 이용하여 계산합니다. 왼쪽에서 오른쪽으로 식을 읽으면서 다음 규칙을 따릅니다.
1. 숫자를 만나면 스택에 넣습니다(Push).
2. 연산자를 만나면 스택에서 숫자 두 개를 꺼내(Pop) 연산한 후, 그 결과를 다시 스택에 넣습니다. (먼저 꺼낸 숫자가 연산자의 오른쪽, 나중에 꺼낸 숫자가 왼쪽 피연산자가 됩니다.)

[계산 과정]
- '3', '4'를 스택에 Push → Stack: [3, 4]
- '*' 연산자 만남 → Pop 4, Pop 3 → 계산: 3 * 4 = 12 → 12를 Push → Stack: [12]
- '5', '6'을 스택에 Push → Stack: [12, 5, 6]
- '*' 연산자 만남 → Pop 6, Pop 5 → 계산: 5 * 6 = 30 → 30을 Push → Stack: [12, 30]
- '+' 연산자 만남 → Pop 30, Pop 12 → 계산: 12 + 30 = 42 → 42를 Push → Stack: [42]
최종적으로 스택에 남은 결과는 42입니다.

문제 34. EAI(Enterprise Application Integration) 구축 유형에서 애플리케이션 사이에 미들웨어를 두어 처리하는 것은? (문제 오류로 가답안 발표시 1번으로 발표되었지만 확정 답안 발표시 1, 3, 4번이 정답처리 되었습니다. 여기서는 가답안인 1번을 누르면 정답 처리 됩니다.)

2) Point-to-point
3) Hub &Spoke
4) Hybrid

정답: 1
해설: EAI(전사적 응용 프로그램 통합)는 기업 내 다양한 애플리케이션 간의 상호 연동을 가능하게 하는 솔루션입니다. 
* Point-to-point (점대점): 미들웨어를 두지 않고 애플리케이션 간에 직접 연결하는 방식입니다.
* **Message Bus (메시지 버스):** 애플리케이션 사이에 미들웨어(엔터프라이즈 서비스 버스 등)를 두어 메시지 형태로 데이터를 통합하고 처리하는 아키텍처입니다.

2) 삽입 이상
3) 삭제 이상
4) 갱신 이상

# 과목: 데이터_입출력_구현 #키워드: 데이터베이스, 정규화, BCNF, 함수_종속

문제 60. 제3정규형 (3NF)에서 BCNF(Boyce-Codd Normal Form)가 되기 위한 조건은?

1) 결정자가 후보키가 아닌 함수 종속 제거
2) 이행적 함수 종속 제거
3) 부분적 함수 종속 제거
4) 원자값이 아닌 도메인 분해

정답: 1
해설: 데이터베이스 정규화의 각 단계별 조건은 다음과 같습니다.
* 1NF (제1정규형): 모든 속성의 도메인이 더 이상 분해할 수 없는 원자값(Atomic Value)으로만 구성 (4번 내용)
* 2NF (제2정규형): 1NF를 만족하고, 기본키가 아닌 모든 속성이 기본키에 완전 함수 종속됨 (부분적 함수 종속 제거, 3번 내용)
* 3NF (제3정규형): 2NF를 만족하고, 기본키가 아닌 모든 속성이 기본키에 이행적 함수 종속되지 않음 (이행적 함수 종속 제거, 2번 내용)
* **BCNF:** 3NF를 만족하면서, 모든 결정자가 후보키인 상태. 즉, **결정자가 후보키가 아닌 함수 종속을 제거**해야 BCNF가 됩니다. (1번 내용)

# 과목: 응용_SW_기초_기술_활용 #키워드: C언어, 비트연산자, 비트논리합

문제 63. 다음 C언어 프로그램이 실행되었을 때의 결과는?

#include <stdio,h>
int main(int argc, char *argv[]) {
    int a = 4;
    int b = 7;
    int c = a | b;

    printf("%d", c);
    return 0;
}

1) 3
2) 4
3) 7
4) 10

정답: 3
해설: C언어에서 `|` 기호는 비트 단위 논리합(Bitwise OR) 연산자입니다. 두 값을 이진수(Binary)로 변환한 뒤, 각 자리의 비트를 비교하여 하나라도 1이면 1, 둘 다 0이면 0을 반환합니다.

* 변수 `b`의 값 7을 4비트 이진수로 변환: `0111`
* 비트 OR 연산 (`0100 | 0111`): `0111`
결과로 나온 이진수 `0111`을 다시 십진수로 변환하면 **7**이 됩니다.
(※ 원문 코드의 `#include <stdio,h>`는 쉼표가 사용된 오타이나, 문맥상 `<stdio.h>` 헤더를 포함하는 것으로 간주하여 풀이합니다.)

public class Operator{

문제 69. 프레임워크(Framework)에 대한 설명으로 옳은 것은?

1) 소프트웨어 구성에 필요한 기본 구조를 제공함으로써 재사용이 가능하게 해준다
2) 소프트웨어 개발 시 구조가 잡혀 있기 때문에 확장이 불가능하다.
3) 소프트웨어 아키텍처(Architecture)와 동일한 개념이다.
4) 모듈화(Modularity)가 불가능하다.

정답: 1
해설: 소프트웨어 프레임워크(Framework)는 개발자가 소프트웨어를 개발할 때 기초가 되는 뼈대(기본 구조, 클래스, 라이브러리 등)를 제공하는 반제품 형태의 소프트웨어입니다. 이를 통해 개발자는 공통된 부분을 처음부터 다시 만들 필요 없이 **재사용(Reuse)**할 수 있어 생산성이 크게 향상됩니다.
[오답 분석]
2번: 프레임워크는 확장이 가능(Extensibility)하도록 설계되어 있습니다. 개발자는 프레임워크 위에 자신의 비즈니스 로직을 추가하여 확장합니다.
3번: 아키텍처는 시스템의 전체적인 구조와 청사진을 의미하는 더 추상적인 개념이고, 프레임워크는 아키텍처를 실제 코드로 구현해 놓은 구체적인 기반 환경입니다. 동일한 개념이 아닙니다.
4번: 프레임워크는 기능 단위로 독립된 모듈들로 구성되어 있어 모듈화(Modularity)가 매우 잘 되어 있습니다.

# 과목: 응용_SW_기초_기술_활용 #키워드: Java, 자바, 제어문, do_while

문제 70. 다음 JAVA 프로그램이 실행되었을 때의 결과는?

public class arrayl {
public static void main(String[] args) {
    int cnt = 0;
    do {
        cnt++;
    } while (cnt < 0);
    if(cnt==1)
        cnt++;
    else
        cnt = cnt + 3;
    System.out.printf("%d", cnt);
    }
}

1) 2
2) 3
3) 4
4) 5

정답: 1
해설: Java의 `do-while` 문은 일반 `while` 문과 달리, 조건을 검사하기 전에 블록 내의 코드를 **무조건 한 번은 실행**하는 특징이 있습니다.
1. `int cnt = 0;` : `cnt`를 0으로 초기화합니다.
2. `do { cnt++; }` : 조건 검사 전 무조건 1회 실행되어 `cnt`는 1이 됩니다.
3. `while (cnt < 0);` : 현재 `cnt`는 1이므로 조건(`1 < 0`)이 거짓(false)이 되어 반복문을 빠져나옵니다.
4. `if(cnt==1)` : 현재 `cnt`가 1이므로 조건이 참(true)이 됩니다.
5. `cnt++;` : `if`문 안의 코드가 실행되어 `cnt`는 2가 됩니다. (else 블록은 무시됩니다)
6. `System.out.printf("%d", cnt);` : 최종적으로 `cnt`의 값인 **2**가 출력됩니다.

# 과목: 프로그래밍_언어_활용 #키워드: C언어, 아스키코드, 문자열, printf

문제 72. 다음 C언어 프로그램이 실행되었을 때의 결과는?

#include <stdio,h>
int main(int argc, char *argv[]) {
    char a;
    a = 'A' + 1;
    printf("%d", a);
    return 0;
}

1) 1
2) 11
3) 66
4) 98

정답: 3
해설: C언어에서 문자는 내부적으로 아스키코드(ASCII) 정수값으로 처리됩니다.
* 대문자 'A'의 아스키코드 값은 65입니다.
* 변수 `a`에 `'A' + 1`을 대입하면 65 + 1 = 66이 저장됩니다. (문자로는 'B'에 해당합니다.)
* `printf` 함수에서 `%d` 포맷 지정자를 사용했으므로, 문자가 아닌 10진수 정수형 값인 **66**이 그대로 출력됩니다. (만약 `%c` 포맷 지정자를 사용했다면 'B'가 출력되었을 것입니다.)
(※ 원문 코드의 `#include <stdio,h>`는 쉼표가 사용된 오타이나, 문맥상 `<stdio.h>`로 간주하여 풀이합니다.)

# 과목: 프로그래밍_언어_활용 #키워드: C언어, 배열, 포인터, 반복문

문제 73. 다음 C언어 프로그램이 실행되었을 때의 결과는?

#include <stdio,h>
int main(int argc, char *argv[]) {
    int a[2][2] = {{11, 22},{44, 55}};
    int i, sum = 0;
    int *p;
    p = a[0];
    for(i = 1; i < 4; i++)
        sum += *(p + i);
    printf("%d", sum);
    return 0;
}

1) 55
2) 77
3) 121
4) 132

정답: 3
해설: 2차원 배열과 포인터 연산에 대한 문제입니다.
* `int a[2][2] = {{11, 22}, {44, 55}};` : 2차원 배열이 메모리상에 `11, 22, 44, 55` 순서로 연속적으로 할당됩니다.
* `p = a[0];` : 1차원 포인터 `p`가 2차원 배열의 시작 주소(값이 11인 위치)를 가리키게 됩니다.
* `for` 반복문이 `i = 1`부터 `i < 4`까지(즉, 1, 2, 3) 순차적으로 실행됩니다.
  - `i = 1`일 때: `*(p + 1)`은 배열의 두 번째 요소인 **22**입니다. (`sum` = 0 + 22 = 22)
  - `i = 2`일 때: `*(p + 2)`는 배열의 세 번째 요소인 **44**입니다. (`sum` = 22 + 44 = 66)
  - `i = 3`일 때: `*(p + 3)`은 배열의 네 번째 요소인 **55**입니다. (`sum` = 66 + 55 = 121)
* 반복문 종료 후 `printf("%d", sum);`에 의해 최종 누적 합계인 **121**이 출력됩니다.

문제 74. 페이징 기법에서 페이지 크기가 작아질수록 발생하는 현상이 아닌 것은?

1) 기억장소 이용 효율이 증가한다.
2) 입·출력 시간이 늘어난다.
3) 내부 단편화가 감소한다.
4) 페이지 맵 테이블의 크기가 감소한다.

정답: 4
해설: 가상 메모리를 관리하는 페이징 기법에서 페이지 크기를 작게 설정하면, 하나의 프로그램을 저장하기 위해 더 많은 수의 페이지로 분할해야 합니다.
따라서 페이지의 개수가 많아지므로, 각 페이지가 메모리의 어디에 적재되었는지 기록하는 **페이지 맵 테이블(Page Map Table)의 크기도 덩달아 증가(커짐)**하게 됩니다. 4번은 반대로 설명했으므로 틀린 보기입니다.
[페이지 크기가 작을 때의 특징]
* 내부 단편화(Internal Fragmentation)가 감소하여 메모리 낭비가 줄어듦 (기억장소 이용 효율 증가)
* 디스크와의 잦은 페이지 교체(Page In/Out)가 발생하여 전체적인 입출력(I/O) 시간이 늘어남

# 과목: 통합_구현 #키워드: 소프트웨어_설계, 모듈, 응집도, Cohesion

문제 75. 다음 중 가장 강한 응집도(Cohesion)는?

1) Sequential Cohesion
2) Procedural Cohesion
3) Logical Cohesion
4) Coincidental Cohesion

정답: 1
해설: 응집도(Cohesion)는 모듈 내부의 구성 요소들이 서로 밀접하게 관련되어 있는 정도를 나타냅니다. 응집도가 강할수록(높을수록) 단일 책임을 가지며 유지보수가 용이해져 품질이 좋은 모듈입니다.
응집도의 세기 순서(강함 $\rightarrow$ 약함)는 다음과 같습니다. (암기 팁: 기-순-교-절-시-논-우)
* 기능적(Functional) 응집도 [가장 강함, 최선]
* **순차적(Sequential) 응집도**
* 교환적/통신적(Communication) 응집도
* 절차적(Procedural) 응집도
* 시간적(Temporal) 응집도
* 논리적(Logical) 응집도
* 우연적(Coincidental) 응집도 [가장 약함, 최악]
따라서 보기 중 가장 강한 응집도는 1번 'Sequential Cohesion(순차적 응집도)'입니다.

문제 76. TCP 프로토콜과 관련한 설명으로 틀린 것은?

문제 77. C언어에서 연산자 우선순위가 높은 것에서 낮은 것으로 바르게 나열된 것은?

㉠ ( )   ㉡ ==   ㉢ <
㉣ <<   ㉤ ||   ㉥ /

1) ㉠, ㉥, ㉣, ㉢, ㉡, ㉤
2) ㉠, ㉣, ㉥, ㉢, ㉡, ㉤
3) ㉠, ㉣, ㉥, ㉢, ㉤, ㉡
