# 5과목 정보시스템 구축 관리

## 1. 소프트웨어 개발 방법론 활용
- **소프트웨어 테일러링 (Software Tailoring)**: 프로젝트의 특성과 필요에 따라 소프트웨어 개발 표준 프로세스, 기법, 도구 등을 커스터마이징(변경 및 최적화)하는 과정.
  - 내부적 요건: 목표 환경, 요구사항, 프로젝트 규모, 보유 기술.
  - 외부적 요건: 법적 제약, 국제 표준 품질 기준.
- **소프트웨어 프레임워크 (Framework)**: 소프트웨어 개발에 공통적으로 사용되는 구성 요소와 아키텍처를 일반화하여 제공하는 반제품 형태의 소프트웨어 시스템.
  - 특징: 모듈화(Modularity), 재사용성(Reusability), 확장성(Extensibility), **제어의 역흐름(Inversion of Control)**.
  - 종류: 스프링(Spring, 자바 기반), 마이바티스(MyBatis, DB 연동), 앵귤러JS 등.

## 2. IT 프로젝트 비용 산정 모델 (Cost Estimation)
- 소프트웨어 개발 규모를 파악하여 필요한 인원, 자원, 기간을 산정하는 기법.

### 하향식 (Top-Down) 산정 기법
- 과거의 유사한 경험을 바탕으로 전문 회의를 통해 산정.
- **전문가 감정 기법**: 조직 내 경험이 많은 두 명 이상의 전문가에게 비용 산정을 의뢰.
- **델파이 기법 (Delphi)**: 많은 전문가의 의견을 종합하여 산정 (익명성 보장, 주관적 편견 보완).

### 상향식 (Bottom-Up) 산정 기법
- 세부적인 작업 단위별로 비용을 산정한 후 전체 비용을 집계.
- **LOC (Lines of Code) 기법**: 각 기능의 원시 코드 라인 수의 비관치, 낙관치, 기대치를 측정하여 예측치를 구함.
  - 예측치 공식: `(낙관치 + 4*기대치 + 비관치) / 6`
- **COCOMO (COnstructive COst MOdel)**: 보헴(Boehm)이 제안. 프로그램 규모(LOC)에 따라 비용을 산정.
  - **조직형 (Organic)**: 기관 내부에서 개발된 중소 규모. 5만(50KDSI) 라인 이하.
  - **반분리형 (Semi-Detached)**: 트랜잭션 처리 시스템 등. 30만(300KDSI) 라인 이하.
  - **내장형 (Embedded)**: 초대형 규모, 미사일 유도 시스템 등. 30만 라인 이상.
- **Putnam 모형**: 소프트웨어 생명 주기의 전 과정 동안에 사용될 노력의 분포를 예상. **Rayleigh-Norden 곡선**의 노력 분포도를 기초로 함. (자동화 도구: SLIM)
- **기능 점수 (FP; Function Point) 모형**: 소프트웨어의 기능 증대 요인(데이터 입력, 출력, 질의, 파일, 인터페이스)에 가중치를 부여하여 산정. (자동화 도구: ESTIMACS)

## 3. 프로젝트 일정 관리 (Scheduling)
- **WBS (Work Breakdown Structure, 업무 분류 구조)**: 프로젝트를 여러 개의 작은 소단위로 분해하여 계층 구조로 표현한 것.
- **PERT (프로그램 평가 및 리뷰 기법)**: 프로젝트의 지연을 방지하기 위해 낙관적인 경우, 비관적인 경우, 정상적인 경우를 나누어 일정을 계산.
- **CPM (Critical Path Method, 임계 경로 기법)**: 노드와 간선으로 이루어진 네트워크. 프로젝트 완수에 필요한 **가장 오래 걸리는 경로(최장 경로)**를 임계 경로로 정의하며, 이 경로상의 작업은 여유 시간이 없다.
- **간트 차트 (Gantt Chart)**: 각 작업의 시작과 끝을 막대그래프 형태로 표시하는 프로젝트 일정표. 시간선(Time-line) 차트라고도 함.

## 4. 소프트웨어 아키텍처 및 스토리지 시스템
- **스토리지 시스템 (Storage System)**: 대용량 데이터를 저장하기 위한 장치와 네트워크의 결합.
  - **DAS (Direct Attached Storage)**: 서버와 스토리지를 전용 케이블로 직접 연결. 확장이 어렵고 케이블 길이 제한이 있음.
  - **NAS (Network Attached Storage)**: 서버와 스토리지를 LAN 네트워크를 통해 파일 서버 형태로 연결. 공유가 용이함.
  - **SAN (Storage Area Network)**: 서버와 스토리지를 연결하는 전용 네트워크(광 채널)를 별도로 구성. 고속 데이터 처리와 확장이 뛰어나지만 구축 비용이 비쌈.

### RAID (Redundant Array of Independent Disks)
- 여러 개의 하드디스크를 배열로 구성하여 하나의 논리적 디스크처럼 사용하는 기술.
- **RAID 0 (Striping)**: 데이터를 여러 디스크에 분산 저장. 속도는 가장 빠르나 장애 시 복구 불가.
- **RAID 1 (Mirroring)**: 동일한 데이터를 두 개의 디스크에 중복 저장. 복구는 완벽하나 용량 효율이 절반.
- **RAID 5**: 패리티(Parity) 비트를 각 디스크에 분산 저장. 1개의 디스크 고장 시 복구 가능. (최소 3개 디스크 필요)

## 5. 정보 보안 및 시스템 보안 구축
- **인증, 인가, 과금 (AAA)**: Authentication(인증, 신원 확인), Authorization(인가, 권한 부여), Accounting(과금, 사용 내역 기록).

### 접근 통제 (Access Control)
- **MAC (강제 접근 통제, Mandatory)**: 주체와 객체의 **보안 등급(레이블)**을 비교하여 시스템이 접근을 통제.
- **DAC (임의 접근 통제, Discretionary)**: 데이터 소유자가 자신의 판단에 따라 사용자 **신분(Identity)**을 기준으로 접근을 통제.
- **RBAC (역할 기반 접근 통제, Role-Based)**: 사용자가 조직 내에서 맡은 **역할(Role)**에 기초하여 접근을 통제.

### 네트워크 침입 탐지 및 차단 시스템
- **방화벽 (Firewall)**: 내부망과 외부망 사이의 패킷을 필터링(IP, Port 기준). 내부의 불법적 행동은 방어하기 힘듦.
- **IDS (Intrusion Detection System, 침입 탐지 시스템)**: 컴퓨터 시스템의 비정상적 사용을 실시간으로 탐지.
  - **오용 탐지 (Misuse)**: 알려진 공격 패턴(시그니처)과 비교하여 탐지.
  - **이상 탐지 (Anomaly)**: 시스템의 평균적인 상태를 기준으로 비정상적 행위 탐지 (오탐률이 높음).
- **IPS (Intrusion Prevention System, 침입 차단 시스템)**: 방화벽과 IDS를 결합하여 탐지된 공격을 능동적으로 차단.
- **VPN (Virtual Private Network, 가상 사설망)**: 인터넷 같은 공중망을 마치 자신의 전용망(사설망)처럼 사용할 수 있게 하는 보안 기술 (터널링, 암호화 적용).

## 6. IT 신기술 동향
- **클라우드 컴퓨팅 (Cloud Computing)**: 인터넷을 통해 IT 자원을 빌려 쓰는 서비스. (IaaS: 인프라, PaaS: 플랫폼, SaaS: 소프트웨어).
- **하둡 (Hadoop)**: 대량의 자료(빅데이터)를 분산 처리할 수 있는 자바 기반의 오픈 소스 프레임워크 (HDFS, MapReduce 사용).
- **고가용성 (HA; High Availability)**: 시스템의 장애가 발생하더라도 서비스가 지속적으로 정상 운영되도록 이중화 등을 구성하는 성질.
- **블록체인 (Blockchain)**: P2P 네트워크를 이용하여 온라인 금융 거래 정보를 참여자(Peer)의 디지털 장비에 분산 저장하는 기술.
